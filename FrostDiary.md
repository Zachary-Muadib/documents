2. 产品概述
	•	一句话愿景：记录值得记录的事
	•	使用场景概述：
	•	日常生活记录：心情、事件、灵感、照片等
	•	自我管理：通过标签和轨迹维度，跟踪习惯、情绪、睡眠、运动等
	•	回顾与反思：按时间、主题、轨迹、标签进行复盘和年度回顾
	•	目标用户画像（主 / 次）：TBD
	•	核心产品特点：高度自由的记录方式
	•	核心价值主张（做什么 / 不做什么）：
使用简洁且结构化的方式，支持用户完成高复杂度的自定义记录。
不加入任何社交、与日记非强相关的功能。
非核心场景通过链接跳转到外部 App。
付费用户可使用更高级的自定义能力。

3. 信息架构（IA）
	•	顶层模块候选：首页（日记）、照片、日历、回顾、我的
默认导航栏位于底部，用户可自定义到顶部显示。
	•	特色功能：轨迹 Trace（用户自定义的分类模块，可用于统计）
	•	全局能力：
	•	快速记录
	•	全局搜索
	•	回顾（自动统计 + 自定义统计）
	•	分享 / 导出

⸻

4. 关键用户流程（Happy Path）
	•	首次启动：
权限声明 → 设定偏好（语言、周起始、默认展示模式等） → 进入首页
	•	自定义：
我的 → 自定义
	•	自定义日记显示模块。
	•	自定义模板与模板推荐。
	•	新建：
首页 → 新增 → 编辑 → 保存或暂存离开
	•	回顾：
按时间或自定义维度生成回顾页面。
	•	检索：
首页 → 搜索框 → 输入关键字 + 使用过滤器 → 结果列表
	•	提醒：
设置提醒频率与时间 → 通知 → 跳转到记录界面
	•	分享 / 导出：
选择条目 / 时间段 → 选择格式 → 预览 → 保存

⸻

5. 界面设计原则与版式
	•	设计关键词：简洁、一致、克制
	•	视觉与排版：
采用 SwiftUI；支持动态字体、深浅色模式；遵循 Human Interface Guidelines。
	•	布局原则：
内容为中心，重要操作放在易触达区域。
	•	状态设计：
空状态、加载状态、无网络状态等逻辑预留。

6. 详细界面规格（按页面）

6.1 首页
	•	展示最新日记 → 旧日记按倒序排列
	•	顶部可开启时间分组（今日、昨日、本周等）
	•	搜索框位于顶部
	•	右上角显示模式切换

三种展示模式：
	1.	备忘录式卡片：
	•	三行文字预览
	•	含图时右侧显示缩略图
	2.	图文模式：
	•	文字最多五行
	•	图片宫格（最多九张）
	3.	顶部大图模式：
	•	顶部大图横向滑动
	•	下方最多五行文字

交互：
点击进入详情；左滑操作（删除等）；下拉刷新。

⸻

6.2 新建 / 编辑
	•	支持标题、正文、图片、标签、轨迹、心情、位置等模块
	•	自动保存草稿
	•	无内容退出则自动删除
	•	可在编辑页删除日记（移动至回收站）

⸻

6.3 时间轴
	•	按年份 → 按月份分组
	•	显示记录数量、代表性日记
	•	支持过滤（按标签或轨迹）

⸻

6.4 日历视图
	•	月 / 周 / 日视图切换
	•	有记录的日期标记
	•	点击日期查看记录列表

⸻

6.5 搜索与筛选
	•	可按关键字、时间范围、标签、轨迹、内容类型进行筛选
	•	支持相关性排序或时间排序

⸻

6.6 标签管理
	•	展示标签、搜索标签
	•	新建、编辑、删除标签

⸻

6.7 统计
	•	时间维度、标签维度、轨迹维度统计
	•	使用简洁图表展示（饼图 / 柱状图 / 折线图）

⸻

6.8 年度回顾
	•	默认展示当前年度
	•	内容包含年度活跃程度、高频标签、轨迹、代表性日记

⸻

6.9 设置（选项逻辑）
	•	外观（深色 / 浅色）
	•	首页展示模式
	•	时间分组开关
	•	默认模板
	•	是否自动获取位置
	•	提醒设置
	•	隐私设置：App 启动验证、多任务界面内容遮挡
	•	数据导出、数据重置

⸻

6.10 回收站 / 归档（逻辑）
	•	删除 → 回收站
	•	回收站支持恢复与彻底删除
	•	自动清理策略（默认 30 天后彻底删除）
	•	归档：
隐藏在主列表外，但仍可搜索与统计

⸻

7. 规则与策略
	•	删除策略：删除进入回收站；彻底删除不可恢复
	•	自动清理策略：回收站内容按周期清理
	•	隐私策略：默认不采集广告数据，不主动引导分享

⸻

8. 本地化与无障碍
	•	支持简体中文与英文
	•	日期格式与周起始遵循系统区域设置
	•	VoiceOver 标签、动态字体、对比度保证

⸻

9. 指标与成功标准（体验层）
	•	首次记录完成率与用时
	•	日均记录完成率、连续记录天数
	•	搜索成功率
	•	复盘转化率（提醒 → 打开 → 完成）

12. 非功能需求（NFR）
	•	性能：滚动流畅、打开速度快
	•	稳定性：编辑不会丢数据
	•	隐私：本地存储、启动验证、多任务预览模糊
	•	兼容性：主流 iOS 版本
	•	可维护性：模块化代码设计

⸻

13. 数据结构与存储策略（概览）
	•	日记 Entry：id、title、content、createdAt、updatedAt、tags、traceOptions、mediaItems、location、mood 等
	•	标签 Tag：id、name、color、sortOrder
	•	轨迹 Trace：id、name、type、options
	•	媒体 Media：id、entryId、type、pathOrIdentifier
	•	提醒 Reminder：id、type、timeOfDay、active

⸻

14. 竞品与差异化要点（概览）
	•	竞品类型：文本日记、图文记录、情绪 App、综合效率工具
	•	FrostDiary 的差异化：
	•	私密、无社交
	•	高度结构化
	•	高自由度自定义
	•	功能边界清晰
